# Constella 用户指南

## 1. 适用范围

- 客户端形态：Web 与 Electron 桌面端
- 典型用户：协作者、房间拥有者、管理员
- 目标：帮助你从首次连接一路到日常协作与插件使用

## 2. 快速开始

### 2.1 准备工作

- 确保后端服务已经启动并可访问
- 准备服务地址，例如 `http://127.0.0.1:3000`

### 2.2 启动客户端

- Web：打开部署地址或本地开发地址
- 桌面安装版：安装并启动应用
- 桌面便携版：解压后直接运行可执行文件

## 3. 登录与房间

- 在首页连接后端
- 登录或注册账号
- 创建、加入与收藏房间

## 4. 画布基础操作

- 平移、缩放与选择节点
- 双击节点进行编辑
- 在节点之间创建连线

## 5. 插件面板

Constella 现在有两个插件入口：

- 设置页 -> 插件
- 房间页 dock -> 插件

插件面板会展示：

- 内置正式节点
- 已安装的外部插件
- 为未来插件市场预留的界面区域

## 6. 插件导入与安装

当前只有 Electron 桌面端支持本地插件安装。

你可以通过以下方式导入插件：

- 将插件文件夹拖入插件拖拽区
- 将 `.constella-plugin` 或 `.zip` 安装包拖入拖拽区
- 点击拖拽区，手动选择本地内容

需要注意：

- `manifest.json` 只是插件目录的入口文件
- 导入 `manifest.json` 的本质是导入它所在的整个插件目录
- 安装过的插件在重启后仍会保留，因为它们会被持久化到本地

## 7. 已安装插件存放位置

Electron 会将插件存放在用户数据目录下：

- 已安装插件内容：`app.getPath('userData')/plugins/installed`
- 导入过的安装包缓存：`app.getPath('userData')/plugins/archives`

## 8. 启用、禁用与卸载

在插件面板中你可以：

- 启用插件
- 禁用插件
- 卸载插件

这些操作会刷新运行时插件注册表，正常情况下不需要整应用重载。

## 9. 导出与资源

- 画布可导出为 JSON、PNG、SVG
- 支持的节点内容可导出为 Markdown、Text、PDF
- 可上传资源并插入到画布中

## 10. 常见问题

### 10.1 插件无法导入

- 检查插件目录中是否存在 `manifest.json`
- 检查 `manifest.json` 中引用的文件是否真实存在
- 如果是压缩包，确认包内结构是插件根目录内容，而不是多包了一层无关目录

### 10.2 为什么重启后插件还在

这是正常现象。插件会被持久化到 Electron 的用户数据目录中。

### 10.3 为什么 Web 端不能安装插件

这是当前设计行为。插件的本地导入与安装仅在桌面端提供。
